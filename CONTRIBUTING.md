# 贡献指南 · 收录标准

Scale 是**策展型**索引，不是有项目就收。提交前请对照收录标准。

## 收录标准（4 选 3 以上才收）

1. **近一年活跃**：`pushed ≥ 当前 - 12 个月`（停更但属经典基础设施的，标 ❄️ 单列）。
2. **拿来即用**：是 Skill / Agent / 工具 / 可直接调用的数据/API，而非纯论文或训练权重（权重类只进「大模型」「数据集」区且需有实用价值）。
3. **法律相关性**：直接服务法律/合规/司法实务。
4. **可核验**：仓库公开存在，能拉到真实 star / license。

❌ **不收**：0 维护的练手 demo、纯 fork、内容农场、无法访问的私有仓、与法律无实质关系的通用工具。

## 每条收录必填字段

| 字段 | 说明 |
|---|---|
| `name` | 项目名 |
| `url` | GitHub/平台链接 |
| `category` | 见 README 分区 |
| `stars` | `gh api` 实测（勿手填估值） |
| `updated` | 最近 push 月份 |
| `license` | SPDX；无 license 标 `无`，copyleft 标 ⚠️ |
| `commercial_ok` | 能否商用：✅MIT/Apache/BSD ｜ ⚠️GPL/AGPL ｜ ❌CC-NC/无 |
| `note` | 一句话执业点评（亮点 + 适用场景） |

## 提交模板（issue / PR 正文）


## 合规底线

只提交**链接**。不要把他人项目的代码、README 全文、或公众号文章正文粘进本仓。见 [COMPLIANCE.md](COMPLIANCE.md)。
